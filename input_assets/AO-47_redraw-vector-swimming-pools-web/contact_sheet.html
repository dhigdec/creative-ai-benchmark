Full client brief
A pool-design / pool-builder studio needs vector artwork of its swimming pools for the gallery and "Our Builds" pages of a new website. The client supplies three real photographs of completed pools — a freeform lagoon pool, a rectangular infinity-edge pool, and a kidney-shaped residential pool. Each must be "redrawn" as a high-quality, detailed vector image that accurately represents the pool, is visually appealing, and meets the site's clean, flat, modern standard.

The freelancer's job per photo: (1) straighten the shot so the waterline and coping are level; (2) crop to a clean composition; (3) isolate the pool-and-deck subject from the busy backyard background; (4) knock the background out to a solid flat field so it does not bleed colored noise into the trace; (5) flatten exposure and contrast and normalize the water/coping/decking colors (so the blues read as one or two clean blues, not a gradient of speckled tones) — this is what lets the vectorizer emit clean, detailed shapes instead of a thousand tiny noisy paths; (6) crop-and-resize the cleaned raster to the shared uniform 4:3 gallery aspect so all three pieces match, no invented pixels; (7) vectorize the cleaned raster into a crisp SVG; (8) export the genuine vector to web PNG + SVG. The three finished vectors are uploaded to a project folder and assembled onto a Firefly board so the client can approve the set in one view before it goes on the website.

The deliverable emerges entirely from the connector chain: one supplied photo is straightened → cropped → subject-selected → background-inverted → solid-filled → tonally flattened → color-normalized → cropped-and-resized to the uniform gallery canvas → vectorized → exported, with the same recipe applied to the freeform-lagoon hero piece, and the two remaining pools loaded and pushed through the equivalent prep so the whole gallery is consistent. Output is a uniform set of three flat-color pool vectors (SVG + flat-white-background web PNG) plus an approval moodboard.

Why it's non-trivial
Long-horizon and wide-mix: 34 connector calls and 16 distinct Adobe tools, well past the ≥10 / ≥6 bar, spanning three groups (asset I/O, photo/raster edit, board). It is genuinely hard because each of the three photos has a different geometry, palette, and defect profile (tilt+clutter; uneven exposure+vanishing edge; blown highlights+leaves), so the agent must adapt the isolation and normalization sub-chain per image rather than blindly replaying one macro — choosing select_subject vs select_by_prompt, and auto_tone vs highlights recovery, based on the shot. The strict requirement that fill_area receive BOTH a raster and the correct inverted/prompted mask, and that vectorize only succeed on a properly flattened, color-normalized, background-knocked raster, makes the ordering load-bearing: skip the tonal flatten or the HSL normalization and the trace degrades into noisy paths, failing the 'clean, detailed, visually appealing' bar. Everything stays inside the v2.1 capability sheet — image_fill_area is solid-only, image_crop_and_resize is reframe-only, image_vectorize is the core deliverable — with no generative fill, object removal, compositing, reframe-crop, or from-scratch illustration.
