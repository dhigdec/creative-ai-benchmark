Willowfen Press is a small independent children's-book publisher about to upload its next paperback, "Pip & the Paper Moon", to Amazon KDP. Our illustrator and layout designer have already built the files, but our last title bounced twice in KDP's preflight (spine width off, one illustration under-resolution, a caption clipped), so before we upload again we want an independent print-QA pass. We are handing off four things: (1) the authored InDesign interior template (pip_interior_template.indd) — a 32-page 8.5x11 in trim book with real InDesign data-merge caption fields (<<page_no>>, <<caption>>, <<alt_note>>) placed on each illustrated spread; (2) the exported interior PDF our designer produced from it (pip_interior_export.pdf, 32 pages); (3) the wraparound paperback cover our designer built to the KDP cover template (pip_cover_wrap.pdf — front + spine + back on one sheet); and (4) the KDP spec sheet we were given (kdp_spec_sheet.pdf) that states the required trim (8.5x11 in), bleed (0.125 in on outside edges), interior safe margin (0.375 in), minimum image resolution (300 DPI), and the spine-width formula for a 32-page white-paper paperback (~0.0725 in). We also include the filled caption data (pip_captions.csv) that maps to the interior template's merge fields. What we need from you: go through the files and tell us, in writing, whether they will pass KDP preflight the first time. Specifically: pull a proof render of the interior template and of the exported interior PDF so we can compare them page for page; inline-preview a representative illustrated spread and the full cover sheet so bleed, safe-margin and spine placement can be eyeballed; read the KDP spec sheet and pull out the exact required numbers; and re-run the caption data-merge into the authored interior template so any long caption that would overset or clip in a text frame is caught (that was one of last time's failures). Then write up a clean, itemized preflight report — one line per check (trim, bleed, margins, spine width, image resolution, caption overset, font embedding, page count) with a PASS / FLAG verdict and a short note, ending with an overall go / no-go and a punch-list of what to fix before upload. Deliver the written report plus the proof renders you pulled. Do NOT redesign or re-lay-out anything — this is a review, not a rebuild; every caption and price stays as supplied.

This document summarizes the key print specifications required for the KDP paperback edition of *Pip & the Paper Moon*.

| Parameter | Requirement |
|---------------------------|-------------------------------|
| Trim size | 8.5 x 11 in |
| Bleed | 0.125 in on all outside edges |
| Interior safe margin | 0.375 in from trim |
| Gutter / inside margin | 0.5 in (for 24-150 pages) |
| Minimum image resolution  | 300 DPI |
| Paper | White |
| Page count | 32 |
| Spine width formula | page_count x 0.002252 in |
| Spine width for 32 pages  | ~0.0725 in |
| Fonts | All fonts embedded |
| Transparency | Flatten all transparency |
| PDF standard | PDF/X compatible (PDF/X-1a, X-3 or X-4) |

Files failing any of these parameters will be rejected by KDP preflight.

This is an Adobe InDesign template for the 32-page interior of the children's picture book 'Pip & the Paper Moon'. The document is configured for US Letter (8.5x11 inches) portrait orientation with a standard facing-pages spread layout.

**Structure & Layers:**
Each of the 32 pages features a full-bleed illustration frame, designed to cover the entire page area. Overlaying these illustration frames on every page is a consistently placed, dedicated text frame for captions.

*   `<<page_no>>`: For dynamic page numbering.
*   `<<caption>>`: For the primary illustration caption text.

These are genuine InDesign merge fields, ready to bind with an external data source for automated content population.

**Dimensions:**
*   Page Size: 8.5 x 11 inches (US Letter)
*   Total Pages: 32

No complex graphic styles or brand-specific elements beyond standard typography.
