Full client brief
A challenge-coin / branding client (automotive-industrial sector) supplies five flat reference images: one primary logo concept and four symbolic coin-face concepts. They are pixelated mock-ups that cannot go to a mint. The task is to convert each into production-grade vector art and deliver a four-color family per mark.

Pipeline per the connector chain: (1) upload all five client rasters to CC storage and preview them; (2) on the primary logo, remove the busy background so only the mark survives on transparency; (3) auto-straighten the de-bagged logo so the mark sits square (coins are minted on a true axis); (4) crop the straightened logo tight to its own bounds so the trace has no dead margin; (5) reframe/resize that cropped mark to a fixed 2000x2000 px square proof canvas via image_crop_and_resize (a gen-free crop-and-scale to the target aspect — no pixels are invented), establishing the exact resolution every variant export inherits; (6) vectorize that clean, square, transparent master into a scalable SVG — this is the editable master vector that also seeds EPS/PDF on packaging; (7) select the mark's silhouette on the 2000x2000 master, then fill it with solid true-black to mint the BLACK variant; (8) on the same selected silhouette, fill it with solid white to mint the WHITE variant; (9) apply a flat color overlay in gold #C8A24B to mint the GOLD variant; (10) apply a monochromatic silver/gray tint to mint the SILVER variant. Then the four coin references are run through the same remove-background -> crop -> vectorize spine so all five marks are coin-ready vector. Finally, assemble every produced vector + variant asset into a single Firefly Board so the client can review the full logo+four-coin family in black/white/gold/silver before mint submission.

Hard constraints honored: no from-scratch illustration (every mark is traced from a client reference), no generative fill / outpaint / object removal / compositing / upscaling; image_crop_and_resize only crops-and-scales the existing mark to the square proof canvas (it never grows or invents canvas), image_fill_area is used only for SOLID color minting (black and white) of an isolated mask, and no generative tool is used at any step. Deliverables map 1:1 to the brief: transparent PNGs, SVG vector source, and black/white/gold/silver color versions, coin-ready and readable at small sizes.

Why it's non-trivial

Difficulty comes from the two parallel chained spines (logo + four coins) plus a three-way variant fan-out from a single cropped master, all of which must stay on-axis and crop-tight so the trace yields mint-grade vectors readable at small sizes.
