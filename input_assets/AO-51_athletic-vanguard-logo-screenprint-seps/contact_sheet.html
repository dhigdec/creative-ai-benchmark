Full client brief
I run a youth/athletic "Vanguard" team and I already mocked up our logo set in ChatGPT, but the files are messy raster PNGs I can't actually use. I need them refined and turned into real production artwork I can hand to a screen printer and also post on social. The combined badge has several elements — the "VANGUARD" wordmark (UC), a subdued American flag, a star icon, chevrons, a soldier head mark, and the outer badge ring. For screen printing each of these has to print as a FLAT spot color, so I need every element reduced to a single clean ink from my military palette — olive drab, khaki, black, and dark grey — with the digital-camo swatch I made available as a fill/accent. Right now the ChatGPT export is dull, slightly crooked, has anti-aliased fuzzy color edges, and the colors aren't consistent enough to separate. So: straighten and tidy it, auto-tone/level it so the palette reads true, then go element-by-element and lock each named mark to one solid spot ink (so my printer gets clean separations), pull a true black plate and a knockout, vectorize the cleaned art into scalable vector, and give me a digital-camo accent swatch as a separate vector too. Final delivery: a genuine layered vector source (.ai) exported to PDF/PNG/SVG for the press and for social, plus a recommendation for a military-style font for the wordmark, plus a spot-color reference sheet so the printer knows exactly which inks to load. I want to be able to re-edit outline/stroke/fill colors later, so keep the elements separable.

Why it's non-trivial
Long-horizon (25 connector calls, 15 distinct Adobe tools) across three groups: raster cleanup/tonal (auto_straighten, auto_tone, vibrance), iterative masked spot-color separation (select_by_prompt/select_subject/invert + fill_area solid repeated per named element — the genuinely hard, error-prone part where each fill must target exactly one element and chain onto the last), and vector/document export + asset packaging. It exercises the honest non-generative path for screen-print seps: fill_area is solid-color ONLY (no gen-fill, no object removal), image_vectorize for raster->SVG, render_vector EXPORTS a genuine authored .ai (never composes one), and font_recommend for the type ask. Difficulty comes from maintaining the spot-lock chain element-by-element without bleeding fills, and from respecting that the .ai must be supplied as an authored input and uploaded rather than generated.
