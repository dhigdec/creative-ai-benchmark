Edit a Dutch real-estate interior to a bright, clean, realistic Streetline-style finish — straighten verticals, recover an HDR-look exposure, fix white balance, pull the blown-out window view back, neutralize one distracting wall outlet, and deliver a high-res JPG.

Full client brief
Kimberley is launching a real-estate photography business in the Netherlands and needs a reliable editing partner for an initial batch of 25-30 listing photos shot for the Dutch/European market (Streetline-style expectations). Using ONE representative hero frame of a living room, run the standard per-image edit recipe end-to-end on the Adobe connector so it can be looped across the whole set with a consistent style:

1) Load the client's supplied hero frame into CC storage (the well-exposed hero interior).
2) Auto-straighten the hero to fix tilted verticals/horizontals (walls must read straight), then crop to clean bounds removing the rotation wedge.
3) Apply auto-tone, then recover exposure / lift the dark portions / tame the highlights to build the bright-but-realistic, HDR-style interior WITHOUT an oversaturated or unrealistic HDR look.
4) Neutralize the warm/cool color cast: correct color temperature for accurate white balance, then gently lift vibrance (no oversaturation) so colors stay natural.
5) Window pull: select the windows by prompt, pull their highlights and exposure down on that mask so the outdoor view (garden/street) reads through the glass instead of blowing out white.
6) Clean one distracting element where necessary: select the small white wall power outlet and solid-fill it to match the surrounding wall color (a legal solid-fill clean-up, not AI object removal).
7) Apply a final consistent, realistic look and retrieve the delivered high-resolution JPG master for client hand-off.

Deliverable: one high-resolution, Streetline-ready JPG of the interior — straight verticals, neutral white balance, bright realistic interior, intact through-window exterior view, distractor cleaned — that defines the repeatable editing style for the remaining images in the batch.

Workflow — 16 tool steps
  1. asset_initialize_file_upload — Begin upload of the client's hero living-room frame into CC storage; returns upload session + target href.
  2. asset_finalize_file_upload — Finalize the hero upload to a usable CC asset URN/presigned URL — the base image the whole edit chain consumes.
  3. image_auto_straighten — Auto-straighten the tilted hero so wall verticals/horizontals read straight — the brief's 'Vertical and horizontal line correction'.
  4. image_crop_to_bounds — Crop away the rotation wedge introduced by straightening for a clean rectangular frame.
  5. image_apply_auto_tone — Baseline auto-tone to set sane exposure/contrast before targeted HDR-look recovery.
  6. image_adjust_dark_portions — Lift the dark/shadow portions to open up the interior — the 'bright but realistic' Streetline look, exposure-blend equivalent on a single frame.
  7. image_adjust_highlights — Pull global highlights down to recover blown areas without an unrealistic HDR halo (brief: 'no unrealistic HDR effects').
  8. image_adjust_color_temperature — Correct the warm/cool cast for accurate white balance — the brief's 'Color correction and white balance adjustment'.
  9. image_adjust_vibrance_and_saturation — Gently lift vibrance only (no saturation push) so colors stay natural — brief: 'Natural-looking colors', 'No oversaturated colors'.
  10. image_select_by_prompt — Select the windows by prompt to build the mask for the window pull.
  11. image_adjust_highlights — On the window mask, pull highlights down so the blown glass recovers toward the exterior garden view — selective masked tonal edit.
  12. image_adjust_exposure — Further reduce exposure on the same window mask to fully resolve the through-glass garden view — completes 'Natural window views (window pull)'.
  13. image_select_by_prompt — Select the small white wall power outlet — the single distracting element to clean 'where necessary'.
  14. image_fill_area — Solid-fill the outlet selection with the wall's neutral color (legal solid-color fill, NOT AI object removal) to neutralize the distractor.
  15. image_apply_preset — Apply one consistent realistic interior look (a neutral bright-clean preset, not a stylized filter) to lock the repeatable per-image style across the batch.
  16. asset_get_presigned_urls — Retrieve a presigned URL for the edited result in CC storage to surface the delivered high-resolution Streetline-ready master JPG for client hand-off.
Why it's non-trivial
Long-horizon and wide-mix: 16 connector calls, all server="adobe", 14 distinct Adobe tools spanning three groups (asset upload/delivery, geometry, tonal/color, masked selection, solid-fill clean-up, preset). Difficulty comes from honest capability mapping under hard constraints: the brief's tempting-but-illegal asks (true HDR merge, AI distractor removal, prompt-based window replacement) are each re-expressed as connector-legal equivalents — single-frame tonal recovery for the HDR look, masked highlight/exposure pulldown for the window pull, and solid-fill for the one distractor — with no generative fill, no compositing, no upscaling, and no Express/Canva. The masked two-stage window pull (select -> highlights -> exposure on the same mask) and the select->fill outlet clean-up require correct mask reuse and order. All steps are [C] headless-confirmed, so the whole chain is autonomously runnable while still reading as a real Streetline-grade per-image editing recipe.
