Full client brief
Aurelia Botanics is launching a premium 3-SKU facial-serum line (Rosehip Renewal, Niacinamide Clarity, Hyaluronic Hydra) and needs retail-ready packaging production. The client provides authored production files (they built the carton dieline in Illustrator and the label layout in InDesign with real merge fields) plus raw brand assets. Your job is the connector-doable production pipeline:

1) Upload all 7 supplied assets (hero product photo, logo, leaf-motif art, carton dieline .ai, label .indd, brand-guide PDF, SKU CSV) into a working folder.
2) Clean up the hero serum-bottle photograph: auto-straighten, auto-tone, then lift exposure and warm the color temperature so the amber glass reads premium and consistent under the studio lighting.
3) Apply a Lightroom "Subject Pop" style preset to make the bottle separate cleanly from the seamless backdrop.
4) Knock the retouched bottle out of its background to a transparent PNG so it can drop onto any mockup backdrop.
5) License and download a marble-surface stock backdrop, and crop/resize it to the exact mockup canvas.
6) Build the botanical side-panel texture: take the supplied leaf-motif art and convert it to a 2-color halftone/duotone in brand green so it prints clean as a spot-style panel.
7) Vectorize the supplied raster logo into a clean SVG for the carton.
8) Export the client's authored carton dieline (.ai) to print PDF + a preview PNG (this is the structural cutting layout — connector EXPORTS it, does not author it).
9) Data-merge the 3 SKU rows from the CSV onto the client's authored label template (.indd with <<product_name>>, <<key_actives>>, <<volume>>, <<batch>> fields) to produce one print-ready label per SKU.
10) Export the merged label document to a CMYK-ready, bleed-correct print PDF.
11) Convert the supplied brand-guide PDF into an editable .indd so the studio can keep the spec sheet in InDesign.
12) Assemble the produced hero PNG, marble backdrop, duotone panel, vector logo, dieline preview and label previews onto a Firefly board as the client-facing concept presentation.

Deliverables: retouched + background-removed hero PNG, marble mockup backdrop, brand-green duotone side panel, vector logo SVG, exported carton dieline PDF/PNG, 3 data-merged label print PDFs, an editable brand-guide .indd, and a Firefly concept board link.

Why it's non-trivial
Long-horizon (18 connector calls, 17 distinct Adobe tools) spanning 4 capability groups: photo/raster retouch (A1), document/layout export + data-merge + PDF convert (A2), asset/stock/board (A4). The 7-step linear hero-retouch chain plus four parallel asset-prep branches that all reconverge on a Firefly board exercise genuine output->input dependency management. It stays strictly inside connector limits: NO from-scratch layout composition (render_vector/render_layout only EXPORT the client-authored .ai/.indd), NO generative fill or by-prompt compositing, image_fill_area not abused (not used at all), and the one [T] step (data-merge) correctly depends on a user-authored .indd carrying real <<fields>> per the v2.1 sheet. The only non-[C] step is the [T] data-merge; everything else is headless-confirmed [C], and there is no [L] local step. Difficulty comes from breadth of distinct tools, depth of the retouch chain, and the discipline of producing real print-prep artifacts (CMYK/bleed PDFs, vector SVG, editable .indd) without crossing into not-available territory.

**Template File:** `aurelia_label_template.indd` (Adobe InDesign Label Template)

**Dimensions:** 70x90mm with a 3mm bleed on all sides.
**Color Mode:** CMYK.

**Structure & Layers:**
The template is structured with distinct layers for text and graphics to facilitate editing and data-merge.
*   **Text Layer:** Contains the dynamic data-merge placeholders.
*   **Graphics Layer:** Holds the static design elements.
*   **Background Layer:** For brand color application.

**Placeholders (Data-Merge Fields):**
The template is authored with genuine InDesign data-merge fields, ready for binding with an external data source:
*   `<<product_name>>`
*   `<<key_actives>>`
*   `<<volume>>`
*   `<<batch>>`

**Typographic Hierarchy:**
A defined typographic hierarchy is applied to the merge fields:
*   `<<product_name>>`: Large, prominent serif typeface.
*   `<<key_actives>>`: Medium-sized, legible sans-serif.
*   `<<volume>>` & `<<batch>>`: Smaller, clean sans-serif for secondary information.

**Graphic Elements:**
*   An accurately positioned image frame is included on the left-hand side, designated for a "leaf side-panel texture" graphic.

**Brand Styling:**
*   **Swatches:** Dedicated brand swatches are pre-defined in the document:
    *   `Aurelia_Green` (CMYK value specified within InDesign)
    *   `Aurelia_Cream` (CMYK value specified within InDesign)
*   These swatches are applied to background elements and accent typography.

**Key Feature:** This is a natively authored InDesign `.indd` file with live, binding `<<fields>>` for data-merge, fully compliant with the `[T]` mode requirement for variable data printing.
