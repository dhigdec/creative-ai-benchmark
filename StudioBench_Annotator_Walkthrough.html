How an annotator scores a task — two complete worked examples

This page follows a creative expert as they score an AI agent's work, end to end. Everything they see and do is shown at every step — so you can understand the whole scoring system from this page alone.

The shape of it: a task is scored in three phases — the input package is validated (Phase 1), a human solves it for reference (Phase 2), then the agent's output is scored across five layers (Phase 3). Two headline numbers fall out: the commission pass rate (would a client accept it?) and human-parity (agent vs the human).

Layer 3 · Verifiers
did it deliver what the brief asked? (mostly code) → K1, K2
Layer 2 · Craft
is it well-made? (expert, Pass/Minor/Major) → K3
Layer 4 · Process
did it work sensibly & honestly? (step-log) → K6, K7
Layer 5 · Review
would a pro ship it? (accept / send-back / scrap)
Layer 1 · Roll-up
the K1–K7 headline, computed from the rest
Marks & weights
Pass / Minor / Major · dealbreaker caps the score

The two examples are deliberately opposite outcomes: Example 1 is rejected (so you see the diagnostic in action), Example 2 is accepted (the clean pass). Both use real benchmark tasks and their real verifier checks.

Legend & the metrics, defined

termmeaning
auto / expertchecked by code vs by a human eye
dealbreaker · mandatory · qualitya dealbreaker caps the capability regardless of craft; mandatory must pass to be accepted; quality grades polish
Pass / Minor / Majorthe craft & process marks — worth 1 / 0.5 / 0; averaged into the score
Commission pass rateshare of outputs a client would accept with no rework (all mandatory verifiers pass, no dealbreaker)
Craft ELOranking from the blind face-offs (which output you'd send the client)
Human-paritybest-of-3 agent run vs the human golden — win / tie / loss
K1–K7the seven capabilities, rolled up from the layers: K1 instruction · K2 asset fidelity · K3 craft · K4 creativity · K5 communication · K6 agentic · K7 honesty

Who scores what (§6.6): code does the objective facts and the step-counts; experts make every judgment and audit a sample of the code. There is no AI judge anywhere. Reliability is measured in the background — 2–3 experts per output, Krippendorff's α per item, hidden gold items, and a senior tie-breaker for genuine taste-splits.
