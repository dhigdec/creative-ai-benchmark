# Data-merge TEMPLATE authoring guide (the ONLY hand-step)

7 of the 10 tasks use Illustrator/InDesign data-merge. The connector runs the merge headlessly,
but the .ai/.indd TEMPLATE must have GENUINE Variables (Illustrator) / Data-Merge fields (InDesign)
— which can only be authored in the desktop apps (you now have them via CC Pro). Author each once;
save into the task folder; then the pipeline merges your generated CSV into it. CSV column names
already match the field names below (the generators were told to.)

## Task realestate-stills — Real-estate weekly listing stills package — 6 scenes from 5-bracket RA
- rider_label.ai — Illustrator artboard 254mm x 152mm (10in x 6in yard-sign rider), 3mm bleed. Author in desktop Illustrator with the Variables panel; bind these as genuine Variables (Dynamic objects), NOT literal text: {{status}} (top banner text object, e.g. 'JUST LISTED'), {{address}} (headline text), {{beds}}, {{baths}}, {{sqft}} (three icon-paired stat text objects), {{price}} (large price text), {{agent_name}}, {{agent_phone}} (footer line), {{brokerage}} (footer). One linked-image Variable {{logo}} for the brokerage logo placeholder. CSV header row must read exactly: status,address,beds,baths,sqft,price,agent_name,agent_phone,brokerage,logo — column names matching the Variable names so document_merge_data_vector binds one artboard per row. Layout: red status banner top, bold address line, 3-up bed/bath/sqft stat row centered, price bottom-left, agent block bottom-right, logo top-right.
