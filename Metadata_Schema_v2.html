StudioBench — metadata schema v2

A granular, crisply-enumerated tag set that replaces the coarse 19. Every tag has a closed value set (ordinals get per-level anchors), a clear set-by rule (auto / derived / human), and a stated slicing use. Grouped into 9 facets; the core subset is the must-have layer, the rest is for deep slicing.

69 tags · 30 core + 39 extended · 9 facets · taxonomy = the 6 distribution axes · distilled from 109 candidates.

Taxonomy axes — how the corpus is organized (the "distribution")

Per the ops-team steer, the corpus is structured by the nature of the task, not by brand. These 6 axes drive the distribution table, balance, and pricing. Every other tag is descriptive — captured for slicing, but not a structuring axis (brand/domain/audience/palette live here).

workflow_nature — The nature of the agent’s job — create / edit / analyze (create = assemble-from-supplied-parts, not generative).
primary_operation — The single dominant operation defining the task's core deliverable, parsed from the golden trajectory's tool sequence via the fixed tool->operation map.
operation_family_set — Set of the four families spanned by all operations in the task, capturing cross-family reach. Derived cache field.
output_modality — Single dominant media class of the primary deliverable. Derived cache field for top-level filtering; never human-set.
horizon_tier — Banded task length derived from est_calls. Derived cache field.
est_calls — Number of connector tool invocations to complete the task (one image_adjust call = 1; a 3-format export = 3 export calls).
