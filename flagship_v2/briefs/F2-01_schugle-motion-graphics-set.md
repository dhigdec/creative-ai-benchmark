## One-line ask
Deliver a coherent German motion-graphics campaign for the ed-tech product Schugle — a 10s silent website-hero loop, a 30s promo/reel, and a 2-minute explainer — built from a branded Express motion template with editable German on-screen text, reframed to 16:9 / 9:16 / 1:1 with per-ratio text placement, accompanied by a clean no-text hero, a separately delivered licensed Stock music bed, a German SRT, and a font/stock/template/music license manifest.


## Brief
Schugle (pronounced "shoe" + the "gle" from "Google") is a German web product that helps teachers create, formulate, structure and revise Foerderplaene (individual pupil support plans) with AI — while protecting sensitive pupil data by replacing real names with anonymised placeholders before anything reaches the AI. The current website is factual and dry; the client wants a coherent visual campaign that makes the idea attractive, memorable and emotionally clear while staying calm, credible, professional and suitable for schools. The mandated narrative arc is "Messy notes -> structured thoughts -> protected placeholders -> AI-assisted formulation -> clear Foerderplan," with the tilted Schugle "g" icon as a recurring motif. The client explicitly forbids generic education clichés (happy teachers, smiling children, chalkboards, graduation caps, cartoon kids, generic "school app" imagery).

Produce THREE related videos that read as one campaign: (1) a 10-second clip that also works as a SILENT website hero/background loop (must read with no voiceover and be visually calm), (2) a 30-second promotional/social/reel version, and (3) a 2-minute explainer that reuses material from the first two. German on-screen text is required and MUST remain editable — the client may swap German copy or add German voiceover later, so text must live as editable layers, never flattened into the only final file. Required German on-screen strings (faithful to the product): "Verstreute Notizen" (scattered notes), "Struktur statt Stapel" (structure instead of piles), "Echte Namen bleiben geschützt" (real names stay protected) with an anonymised placeholder like "[Schüler:in A]", "KI-gestützte Formulierung" (AI-assisted formulation), and "Klarer Förderplan. Weniger Schreibarbeit." (clear support plan, less paperwork).

Constraints and deliverables: master the campaign at 1080p, 30fps, H.264 MP4. Deliver each of the three videos in three aspect ratios — 16:9 (1920×1080, website hero), 9:16 (1080×1920, reel/Stories) and 1:1 (1080×1080, feed). Because the in-connector reframe tool (video_resize) is SUBJECT-AWARE, not text-aware, the German on-screen text is NOT baked into a single master before reframing; instead the brand motion is reframed first and the German text is placed PER-RATIO afterward (as an editable text overlay) so it stays legible and in-frame in every ratio. Provide a clean NO-TEXT version of the 10s hero loop (silent, for the website background) — generated from a separate text-free brand animation, not by stripping text off the text version. Deliver the licensed Stock music bed as a separate WAV stem (the brief: "separate audio files/tracks for music"); loudness normalization to roughly -16 LUFS is performed as an out-of-band local audio step (ffmpeg loudnorm) because no connector tool normalizes loudness — the 30s and 2-min carry the music, the 10s hero stays silent. If captions are shown, supply a German SRT (authored locally from the campaign copy, then uploaded). Finally deliver a license manifest (CSV/MD) listing every font, Stock asset (with license id), Express template id and music track used, with license terms. The visual style must echo the calm, trustworthy German-school tone of the named reference (foerder-fokus.de) without copying it, and must be grounded in how the real product actually flows (see the supplied reference tutorial clip).

**Iterative chaining:** Genuinely sequential, not parallel. (1) The supplied reference tutorial is summarised (step 3, [A]) and that beat list shapes the template search (4), the storyboard, and the Stock-music sizing (8) — one content output feeding several downstream decisions. (2) Brand-identity outputs feed forward: the traced g-icon (1) and recommended font (2) flow into both animate branches (6/7) and the per-ratio text render (11). (3) The core pivot is the Express pipeline split into TWO honest branches: search_design (4) → animate_design TEXT-FREE (6) = the clean brand motion, and search_design (4) → fill_text (5) → animate_design TEXT (7) = the text master. The clean motion (6) is the only input to the subject-aware reframe (9) — safe because it has no text — while the German copy is (re)placed per-ratio via document_render_layout (11) and composited locally (15), so text never depends on autoreframe keeping a lower-third in frame. (4) The licensed music (8) is delivered as a separate stem and loudnormed only in the local mux (15). (5) The 2-min explainer is built deterministically in the local concat (15) by re-sequencing reused 10s+30s segments in beat order — quick_cut (13) is kept only as an optional best-effort highlight, never the sequencer. (6) The SRT and manifest are authored locally (16) from the same German data (foerderplan_sample_data.json), the step-4 template id, the step-8 Stock license id and the step-2 font, then everything is uploaded together into the step-12 folder tree (14). Inputs are heavily reused: the g-icon spans 1/2/6/7, the German copy data spans 5/11/14/16, the reference summary spans 3/4/8, the wordmark spans 2/6/7/11.


**Why it's hard:** HARD on every axis a long-horizon agent struggles with, now grounded in the corrected capability reality. (a) Long horizon: 16 dependent steps (14 connector calls across 4 connector groups + 2 local glue steps) with no shortcut — skipping the summarize de-grounds the storyboard, skipping the icon trace breaks brand continuity, and the clean vs text branches must be kept distinct. (b) Heavy state tracking: the agent must carry the 5-beat order, the exact German strings (umlauts + 'Schüler:in' colon-form), the chosen font, the template id, and the Stock license id all the way to the final manifest — while keeping the clean no-text hero (step 6 → silent trim) and the text version (step 7 + per-ratio overlays) as two separate branches. (c) Exact-spec pressure: nine video files at three precise resolutions (1920×1080 / 1080×1920 / 1080×1080), 30fps, ~10/30/120s durations, a SILENT loop vs scored versions, music near -16 LUFS — and crucially the agent must KNOW these exact-length/codec/loudness guarantees come from the local ffmpeg step, not from animate_design or a non-existent loudness connector. (d) Capability discipline: the agent must NOT misuse media_enhance_speech on music, must NOT rely on subject-aware video_resize to preserve baked text (text goes on per-ratio after reframe), must NOT treat quick_cut as a deterministic sequencer, and must author the SRT/CSV locally — every one a trap the original spec fell into. (e) Faithfulness traps: the brief forbids generic-school clichés, demands editable text + a complete license manifest, and the agent must correctly NOT add voiceover (client supplies German VO later). (f) Execution-mode awareness: the honest path mixes [C] headless steps, [W] Express widget steps that need a user template pick, [A] async video/summarize widgets, and [L] local glue — so the task is product-doable but explicitly NOT autonomous-headless, and a grader who assumed full headless execution would be wrong.
